### Working Agreements

**Code Quality**
- All code changes must go through a pull request (no self-merging)  
- Use clear variable names and follow Django conventions  
- Branch naming format: `feature/<issue>-<short-desc>`  
- Example commit messages:  




**Testing & Documentation**
- Manual testing is required before merging  
- Update README for any setup or environment changes  

**Work Norms**
- Collaborate asynchronously via GitHub and WhatsApp  
- Keep discussions respectful, concise, and constructive  
- Thanksgiving week = reduced availability; adjust sprint scope accordingly  

**Conflict Resolution**
1. Discuss directly between involved members  
2. If unresolved, raise to the Scrum Master  
3. If persistent, bring up during the next team meeting or retrospective  

**Pair Programming**
- Recommended for complex tasks such as deployment or scraper logic  
- Suggested pairs: Louis + Sachit (backend), Louis + Parm (UI)
